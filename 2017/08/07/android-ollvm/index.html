Android Native Obfuscator-llvm

Android
  • Native
  • Obfuscator-llvm

此次全国大学生信息安全竞赛之中坑就坑在要在比赛之前提供题目，然后选择了出安卓题，就遇到了很多的坑点

最值得学习的是在Android native 加入使用了 ollvm，

找了很多的文章，都是先要在linux下面将Android的Native 实现编译完成之后然后，再加到Android Studio下

https://github.com/obfuscator-llvm/obfuscator

编译的时间是很漫长的，最后编译了几次之后，选择了放弃。

之后遇到了一个大佬，说windows下也是可行的。

然后我就开始了尝试windows之下编译 obfuscator-llvm 之旅

找着找着就找到了 上海交大的 https://github.com/GoSSIP-SJTU/Armariris 混淆模块。

之后下载下来之后，按照官方的说明。使用

mkdir obf
cd obf
clone git@github.com:gossip-sjtu/Armariris.git
cmake -DCMAKE_BUILD_TYPE:String=Release ./Armariris
make -j4

cmake 命令 的话 我这边安装了 vs2017 直接可以执行的。

make -j4 的话 因为上一步之后目录下会生成 windows 程序的项目文件， 直接使用 Vs2017编译，编译的时间比较久。CPU、内存消耗也比较大。

之后能够编译出 bin/目录 下面有许多exe文件，还有lib文件夹，里面包含许多的头文件

然后 将bin文件夹内的 

clang++.exe
clang.exe
clang-format.exe

替换 sdk-path\ndk-bundle\toolchains\llvm\prebuilt\windows-x86_64\bin 内的文件

之后可以直接编译，会出现问题，就是 丢失头文件。

报错说缺什么文件就补什么文件。

我这边 sdk\ndk-bundle\platforms\android-15\arch-x86\usr\include 缺少了 stddef.h,stdarg.h两个文件。

如果arm,mips也需要的话在相应的include文件夹内也要复制进去相对应的文件。

缺少的文件可以去相对应编译的 \build\Release\lib\clang\3.9.1\include 目录之下寻找，只要导入即可。

然后就能够编译成功了

之后如果要使用的话，需要在app的 build.gradle 的 cmake cppFlags 中加入 功能

编译时候添加选项开启字符串加密

-mllvm -sobf
开启控制流扁平化

-mllvm -fla
开启指令替换

-mllvm -sub
指定随机数生成器种子

-mllvm -seed=0xdeadbeaf

这样就成功加好了OLLVM ，其中当前版本的 加密字符串可能会有一点小小问题，如果自己又对字符串进行一定的处理的话，会导致字符串的混乱（期待他们实验室的完善）。

目前其实在反控制流平坦化方面也做的不错了 ，这个是基于x86情境下的
https://github.com/SCUBSRGroup/OLLVM_Deobfuscation

还有freebuf上有文章
http://www.freebuf.com/articles/terminal/130142.html
