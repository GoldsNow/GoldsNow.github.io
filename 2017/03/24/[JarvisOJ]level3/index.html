JarvisOJ level3

Writeup
  • Jarvis oj
  • pwn

这个题目虽然和前几个的结构差不多，但是减少了 system 和 /bin/sh ，而是给了一个libc.so 需要自己从里面找到地址

首先checksec下

gdb-peda$ checksec
CANARY    : disabled
FORTIFY   : disabled
NX        : ENABLED
PIE       : disabled
RELRO     : Partial

发现堆栈不可执行。

这里就需要用到write函数来leak地址了。

所以ret所对应的eip为 plt_write = 0x08048340

使用write函数需要有三个参数，就是ssize_t write(int fd,const void *buf,size_t nbytes)

fd = 1 的时候是使用标准输入

因为需要构造write来使得 read的真实地址泄漏
所以 我们使用write的三个参数为1 got_read len(got_read)=4

stack
write
addr_fun
1
read_got
4

这样子就可以leak read的真是地址，并且还能够在运行一次fun函数

接下来就是找system的真实地址了。在libc里找

system = libc_system - libc_read + true_read

/bin/sh 可以通过 strings -a -t x ./level3 | grep /bin/sh 来找到地址，或者直接通过ida等工具
